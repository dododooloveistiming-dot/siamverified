# Owner outreach

Prioritized invitations for venue owners to claim their verifiedthai.com listing.
Generated by `scripts/build_outreach.py` — **never commit** (contains owner email addresses).

## What's in here

- `queue.json` — all 243 ranked contactable venues (read-only program input)
- `queue.csv` — same, Google Sheets-friendly
- `drafts/{niche}/{rank}_{slug}.txt` — per-place email draft (subject + body)
- `batches/top50_{niche}.csv` — first 50 per niche, staged send

Regenerate any time after re-scraping signals:

```
python scripts/build_outreach.py
```

## Priority score

| Signal | Boost |
|---|---|
| `same_domain` email (contact@place.com) | +30 |
| `personal` email (gmail/yahoo) | +15 |
| Has Klook product (pays ~25%) | +25 |
| Veteran ≥10y wayback | +10 |
| Review in last 30d | +10 |
| Pro email infra (Google/MS365/...) | +5 |
| `trust_score / 5` | +0–20 |

Excluded: already-partner, email tier `other`, trust < 30.

## Sending — operational rules

These exist because cold mass-mail to scraped addresses gets you blacklisted.

1. **Send from a real human-named address** (not `noreply@`). Replies should
   land in an inbox someone reads.
2. **Batch size ≤ 50/day, ≥ 24h between batches.** Anything bigger triggers
   spam classifiers. Burn the domain reputation and you can never recover.
3. **Top-down by rank.** Highest-priority same_domain venues first — those
   replies are signal for whether the template works before scaling.
4. **Watch for bounces.** Hard bounces ≥ 5% in a batch → stop, the scrape is
   stale or the template tripped a filter.
5. **Honor `REMOVE` replies within 24h.** Set partner=false, delete the
   place from `places.json` source list, or add to an exclusions file.
6. **Track sends.** Move sent drafts to `sent/{YYYY-MM-DD}/` so re-runs of
   the script don't re-queue them. (Future: have the script read `sent/` and
   skip those place_ids automatically.)

## What the template promises

- 0% commission on direct inquiries (vs. Klook's ~25%)
- Free claim → manage hours, photos, services, Korean/English notes
- Listing already built from public Google + website data
- "REMOVE" reply → takedown within 24h

The Klook line only appears for venues where `per_place_klook.json` shows
actual products — anchors the value prop in their existing pain.

## After the first 50

If reply rate < 2% → template's not landing, rewrite before continuing.
If reply rate ≥ 5% → scale to next 50, then 100, etc. Track per niche —
cooking schools respond differently than muay-thai gyms.
